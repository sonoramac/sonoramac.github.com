Distributed Notifications

Sonora posts distributed notifications for when the currently playing track changes and also when the player state changes. Add the following notification name constants to your code and use NSDistributedNotificationCenter to register for these notifications:


static NSString* const kSonoraDistNotificationPlayerStateChanged = @"com.iktm.Sonora.stateChanged";
static NSString* const kSonoraDistNotificationCurrentTrackChanged = @"com.iktm.Sonora.trackChanged";

Neither of these notifications contain a userInfo dictionary due to Apple's sandboxing restrictions. Instead, Apple Events should be used to read the current track information and player state upon receiving a notification. Read below for more information on this.

AppleScript Support

Sonora has basic AppleScript support for tasks such as reading current track information, controlling music playback, and setting player attributes. To see full documentation on Sonora's AppleScript support, open the scripting dictionary for Sonora from AppleScript Editor (File > Open Dictionary). Below are a few downloadable example scripts:

Using ScriptingBridge

The recommended method for interfacing with scriptable applications from Cocoa applications is to use the ScriptingBridge framework. Read the usage documentation to learn how to integrate Sonora support into your Cocoa apps.
